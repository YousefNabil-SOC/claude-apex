# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Claude Pantheon, please report it responsibly:

1. **Do NOT** open a public issue
2. Create a **private security advisory** on GitHub:
   - Go to the repository's Security tab
   - Click "Report a vulnerability"
   - Provide details about the vulnerability

## What Counts as a Security Issue

- API keys, tokens, or credentials accidentally committed
- Personal data exposure in any file
- Installer scripts that could damage user systems
- Hook scripts with command injection vulnerabilities
- Configuration files that weaken Claude Code security

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 1 week
- **Fix**: As soon as possible, depending on severity

## Scope

This policy applies to the claude-pantheon repository and its installer scripts. Third-party integrations (OMC, PAUL, CARL, etc.) have their own security policies.
